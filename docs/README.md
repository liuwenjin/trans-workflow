## 为什么构建 Agent 工作流越来越重？

今天市面上的 Agent 编排平台正在变得前所未有的臃肿：

* ❌ **环境劝退**：动辄需要 Docker 镜像、复杂的 Python/Node 环境与多数据库组合（MySQL、Redis、Milvus...）。
* ❌ **黑盒困境**：中间过程难以干预，无法实时调整生成逻辑，调试成本极高。
* ❌ **概念冗余**：充斥着大量晦涩的节点和连线规则，学习曲线陡峭。
* ❌ **商业限制**：免费版处处受限，企业版费用昂贵。

但对于独立开发者、内容创作者以及敏捷团队来说：

> **真正需要的，只是一个简单、透明、直观且完全可控的 Agent 协同工具。**

Trans-Workflow 正是为此而生。

---

# 什么是 Trans-Workflow？

Trans-Workflow 是一款以 Agent 为核心节点、基于纯 HTML 的轻量级可视化工作流引擎。

> 💡 **能力基座**：目前节点深度结合 TransWeb 生态。你可以打开 [agent.transweb.cn](https://agent.transweb.cn)，通过纯自然语言（意图驱动）直接生成你需要的 Agent 运行模块。

通过可视化画布，你可以将多个 AI Agent 像搭积木一样串联，让它们各司其职、高效协同。

### 核心特质

* 📦 **单文件形态**：无环境依赖，一个 HTML 文件即是一切。
* 👁️ **全流程可见**：中间结果不仅“可见”，而且“随时可编辑”。
* 🔄 **精细化控制**：支持节点级回溯与重触发，终结“一键重跑”的噩梦。
* 🔓 **绝对掌控**：完全开源，无节点限制，100% 私有化运行。

# 核心能力

## 1. 多 Agent 异步/同步串联

支持多个 AI Agent 按特定序位协同工作。前置节点的输出自动解构并注入为后置节点的输入，无需复杂配置即可构建完整的生产链路。

```text
 选题 Agent ➔  大纲 Agent ➔  内容 Agent ➔  润色 Agent

```

## 2. 过程透明与人工干预（Human-in-the-Loop）

传统平台视中间流转为“黑盒”，调试成本极高。Trans-Workflow 支持**实时查看节点执行状态**与输出内容。你可以随时暂停流程、手动修改当前的中间结果，再让后续 Agent 基于修改后的数据继续运行。

> **真正实现 Agent + Human 的无缝协同。**

## 3. 节点级回溯与重触发

在长文本或复杂内容生成场景中，如果第 4 步（如：润色 Agent）效果不满意，你**无需从头重新跑前 3 步**。直接定位到该节点，调整 prompt 或手动干预后单独**局部重跑**。

```text
[✔] 1. 选题分析
[✔] 2. 结构大纲
[✔] 3. 初稿内容
[➔] 4. 风格润色 ─── (不满意？一键重置此节点，即刻局部重新生成)

```

## 4. 动态自由编排

工作流随业务需求即时调整。支持画布内**动态新增、删除、拖拽调整顺序、修改配置**，并可一键导出为全新的工作流配置文件。**所见即所得。**

## 5. 自动持久化与数据主权

所有的执行痕迹、节点状态、最终产物均在本地自动实现持久化保存。支持历史记录回溯、二次加工与数据导出，**数据资产 100% 归用户所有**。

## 6. 颠覆性的「单文件部署」

这是 Trans-Workflow 最具差异化的设计。整个系统被极致地浓缩进**一个 HTML 文件**中：

```text
一个 HTML 文件 = 编辑器 + 工作流引擎 + 运行环境

```

彻底告别 Docker、MySQL、Redis 及复杂的后端运行时。将其上传至任意 Web 服务器（甚至直接双击本地打开），即可拥有一套完整的 Agent 协同系统。
