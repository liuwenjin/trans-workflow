# Agent 节点

在 Transweb 工作流中，Agent 节点是执行核心任务的智能单元。您可以根据业务需求，随时调整节点的名称以及底层驱动的大模型算力。

## 修改节点名称

为了让工作流更具可读性，建议根据 Agent 承担的具体角色（如：翻译官、代码审查员）来命名节点。

**操作步骤：**

添加可选 Agent 应用后，可以在“修改 Agent 列表”的对话框中，修改 Agent 节点的名称。

1. 点击“设置”-> "修改 Agent 列表"

![点击修改菜单](https://transweb.cn/workflow/docs/images/agent001.png "点击修改菜单")

2. 在弹出的对话框中，定位到目标 Agent，直接修改其名称并保存。

![点击修改菜单](https://transweb.cn/workflow/docs/images/agent002.png "点击修改菜单")

## 配置 AI 算力参数

Transweb 支持为每个 Agent 节点灵活切换底层大语言模型（LLM）。您可以直接选用国内外主流的优秀模型，也可以配置自己注册的算力平台 API Key。

**操作步骤：**

1. 在导航菜单，点击需要配置的 Agent 节点名称。

![打开 Agent](https://transweb.cn/workflow/docs/images/agent003.png "打开 Agent")

2. 点击右上角菜单 「修改」 -> 「配置 AI 算力」，即可选择或切换您心仪的大语言模型。

![配置 AI 算力](https://transweb.cn/workflow/docs/images/agent004.png "配置 AI 算力")


## 自定义算力安全与隐私保护机制

当您使用自定义算力供应商并填写私有 API Key 时，Transweb 提供企业级的安全隔离保护：

### 🔒 API Key 核心防护机制
- 权限严格隔离： 
只有该 Agent 应用的**原生创作者（所有者）**有权读取和配置 API Key。未登录用户或非所有者账号，绝对无法通过任何接口请求到该 Key。

- 运行期防截获： 
其他用户在调用或执行该 Agent 工作流时，API Key 会在后端安全合规地解密调用，前端无法通过反编译、网络抓包或拦截消息获取到 Key 的明文。

### ⚠️ 注意（本地缓存提示）：
在配置并确认自定义 AI 算力参数时，为了保障开发流程的连贯性，在生成 Agent 应用代码前，API Key 可能会临时保存在您的本地浏览器缓存中。如果您在公共设备上操作，或对本地缓存安全性有极端要求，请在结束操作后清除缓存，或优先使用系统托管的公共算力资源。
