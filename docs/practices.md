# 最佳实践

在人机协同与 “Vibe Coding” 的时代，真正的生产力不来自于复杂的架构和冗余的配置，而来自于极致的专注与闭环。本最佳实践旨在打造一套单文件、零部署、具备节点级回滚能力的极简 AI 工作流规范。通过抹去繁琐的语法与环境成本，我们让开发者能够将全部精力聚焦于“意图表达”，实现从自然语言到可执行软件的无缝流转与快速迭代。

---

## 💡 核心概念与持久化机制说明

在正式构建工作流前，必须清晰理解平台的**无状态运行模式**及数据持久化边界，以防配置丢失：

* ❌ **工作流参数（内存实时状态）**：在 Web 端配置的拓扑结构与节点参数均临时运行在客户端的内存中。**线上平台不提供自动保存，只有在手动执行“下载导出”后，配置才会以 `.html` 文件形式持久化至本地。**
*  **Agent 应用（云端持久化）**：无论是通过 AI 智能生成还是自选创建的每一个 Agent 原子应用，均会自动实时持久化保存于 Agent 基座系统的后台数据库中。
* 🌐 **Agent 基座官方访问地址**：[agent.transweb.cn](https://agent.transweb.cn)

## 一、 全生命周期配置流程最佳实践

### 1. 准备与创建可选 Agent 应用节点
在组装工作流链条前，保证所需的 Agent 应用已成功导入当前项目的备选列表中。请先打开 Agent 工作流界面（[transweb.cn/workflow](https://transweb.cn/workflow)），并遵循以下标准作业程序（SOP）：

| 步骤 | 操作阶段 | 具体操作路径 | 标准输出与交付物说明 |
| :---: | :--- | :--- | :--- |
| **1** | **唤起配置窗口** | 点击工作流界面右上角的 **「新增」** 按钮。 | 系统成功弹出 *“新增可用 Agent”* 交互对话框。 |
| **2** | **导入节点<br>(方式 A：AI生成)** | 在对话框顶部的自然语言输入框中，深度描述你的 Agent 业务需求，点击 **「开始生成」**。 | 等待 1~3 分钟，AI 智能引擎将自动完成节点代码与逻辑构建。预览无误后点击 **「确认添加」**。 |
| **3** | **导入节点<br>(方式 B：自选已有)**| 在对话框下方的企业级 Agent 应用资产列表中，直接浏览、检索并点击目标 Agent 实例。 | 进入 Agent 详情页，核对版本参数无误后，点击 **「确认添加」**。 |
| **4** | **校验备选列表** | 循环上述导入方式，直到当前工作流所需的全部原子节点均加入备选库。 | 点击节点下拉列表时，所有关联的 Agent 应用节点正常高亮呈现，无缺失项。 |

### 2. 配置工作流节点拓扑结构
在**工作流 Agent 节点配置核心区域**，开发者应按照标准的业务逻辑流水线对 Agent 进行串联组装：

1. **配置拓扑首节点**：聚焦并点击左侧起始输入框，在弹出的下拉备选列表中选择一个 Agent 应用作为整个工作流的初始输入触发节点。
2. **配置后续依赖节点**：依次点击右侧关联的输入框，根据业务上下游的数据依赖关系（如：上游节点的输出作为下游节点的输入参数），在下拉列表中选择对应的 Agent 应用进行链式绑定。

> 📌 **完成标志**：当整条业务流水线上的所有 Agent 节点全部完成精确选择与顺序串联后，当前工作流的逻辑拓扑即构建完成。

### 3. 导出与持久化工作流快照
由于线上的 Trans-Workflow 默认采用轻量化的**无状态运行模式**（平台不自动实时缓存、不代存用户的任何配置参数结构），完成配置后切记手动保存：

* **标准导出动作**：点击界面右上角的 **「下载导出」** 图标按钮。
* **持久化产物**：系统会即时打包当前内存状态，并自动生成下载一个包含完整拓扑配置的最新 `.html` 架构存档文件至本地电脑。

## 二、 开发者进阶：零部署架构与企业级二次开发

> 🛠️ **生产级演进提示：**
> 如果您或您的团队在复杂业务场景下，对工作流系统提出了更高阶的要求，例如：**实时状态的服务端缓存、多版本历史配置比对、企业节点异常自动回滚**等，建议脱离线上无状态模式，转向开源生态进行二次开发。

### 1. 零部署架构 (Zero-Deployment Architecture)
建议直接克隆或下载 Trans-Workflow 的开源代码仓库。该项目在架构设计上原生支持**单文件、免复杂依赖环境**快速拉起，可无缝嵌入现有的 CI/CD 流水线。

### 2. 状态持久化改造
基于开源架构，企业团队可快速进行以下扩展：
* **数据库对接**：改造前端无状态代码，对接企业内部的 `MongoDB`、`PostgreSQL` 或 `Redis`。
* **高可用升级**：将内存级工作流参数升级为服务端自动化持久化，支持全路追踪、多用户协作及历史版本的一键回滚。
